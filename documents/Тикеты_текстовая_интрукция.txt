Тикеты:
Начальная страница тикетов 
На начальной странице отображается интерфейс управления тикетами. 
Список команд: 
• Создать - создать новый тикет. 
• Создать подзадачу - создать тикет подчиненный текущему.
• Настроить колонки - установить видимость и порядок расположения 
колонок (включая дополнительные сведения).

Настройки отбора 
• Отбор: Все - отображать все тикеты согласно правам. 
• Отбор: Выполняю - отображать тикеты, в которых текущий 
пользователь является исполнителем. 
• Отбор: Поручено - отображать тикеты, в которых текущий пользователь 
является автором. 
• Отбор: Статусы - отбор по статусам. 
• Отбор: Исполнитель - отбор по выбранному исполнителю. 
• Отбор: Проект - отбор по выбранному проекту (отображается при 
включенной опции в настройках).

В зависимости от состава отборов список может отображаться деревом 
(иерархия по подзадачам) или плоской таблицей. 
Условия для дерева: 
• Отбор по статусам не включен. 
• Пользовательские отборы пусты. 
• Отбор по типу исполнения "Все" + отбор по "Автору" или 
"Исполнителю". 
• Отбор по типу исполнения "Выполняю", отборы по "Автору" и 
"Исполнителю" отключены. 
• Отбор по типу исполнения "Поручено", отборы по "Автору" и 
"Исполнителю" отключены. 
Во всех остальных случаях выводится плоская таблица. 
В меню "Еще - Отображать списком" включает отображение списком, для 
дерева при большом объеме данных может быть замедление формирования.

Процесс подачи заявки 
При создании нового тикета обязательно заполняется тема задания. 
 
 
Список участников заполняется автоматически, при необходимости 
можно добавить пользователя своей организации 
 
Настройки уведомлений 
Пользователя можно назначить куратором, который имеет такие же 
привилегии над тикетом, как и автор. 
Назначать кураторов может автор. 
Параметры уведомлений настраиваются индивидуально для Тикета. 
Заполнение описания 
По кнопке «Описание» открывается форма редактирования, в которой 
необходимо описать поставленную задачу 
В область «Перетащите файлы и ссылки» можно прикреплять 
необходимые файлы 
После заполнения темы и описания тикет необходимо записать и 
закрыть по соответствующей кнопке. 
Отслеживание заявки 
Поле «Комментарий» предназначено для ведения диалога между 
участниками тикета. 

Статусы тикетов 
При создании у тикета статус «Новый» 
Исполнитель может принять его в работу или при необходимости 
отправить на согласование. Заданию присваивается статус «Требует 
согласования», и у автора появляется кнопка «Согласовать», после чего 
исполнитель переводит тикет в статус «Выполняется» 
После выполнения работы ставится статус «Проверяется» 
Автор может завершить проверку и перевести тикет в статус 
«Проверено», или при необходимости отправить на доработку 
После подписания ЛУРВа по кнопке «Подтвердить выполнение» тикет 
закрывается и ему присваивается статус «Выполнено»
